# mcp-kaiten

[English version](README.md)

MCP-сервер для **Kaiten** — карточки, списания времени, доски, комментарии, пользователи.

Подключите Cursor, Claude Desktop или любой MCP-клиент к вашему Kaiten.

Запуск одной командой: `npx -y mcp-kaiten`.

---

## Быстрый старт

### 1. Получите API-токен

1. Откройте ваш Kaiten (например, `https://your-domain.kaiten.ru`)
2. Перейдите в Профиль → API Key
3. Создайте новый токен и скопируйте его

### 2. Добавьте в Cursor / MCP-клиент

```json
{
  "mcpServers": {
    "mcp-kaiten": {
      "command": "npx",
      "args": ["-y", "mcp-kaiten"],
      "env": {
        "KAITEN_API_TOKEN": "your-api-token",
        "KAITEN_URL": "https://your-domain.kaiten.ru"
      }
    }
  }
}
```

Сервер запускается автоматически, когда MCP-клиент к нему обращается.

## Аутентификация

Kaiten использует API-токены для аутентификации. OAuth не поддерживается API Kaiten.

1. Перейдите в профиль Kaiten → API Key
2. Создайте и скопируйте токен
3. Укажите `KAITEN_API_TOKEN` в конфигурации MCP

### Переменные окружения

| Переменная | Обязательна | Описание |
|------------|-------------|----------|
| `KAITEN_API_TOKEN` | да | API-токен (Bearer) |
| `KAITEN_URL` | да | URL инстанса Kaiten (например, `https://your-domain.kaiten.ru`) |
| `KAITEN_DEFAULT_SPACE_ID` | нет | ID пространства по умолчанию для поиска карточек (если `spaceId` не указан) |
| `KAITEN_REQUEST_TIMEOUT_MS` | нет | Таймаут HTTP-запроса в мс (по умолчанию: `10000`, диапазон: 1000–60000) |
| `KAITEN_CACHE_TTL_MS` | нет | TTL кэша пространств/досок/пользователей в мс (по умолчанию: `300000`, 0 — отключить) |

## Детализация ответов

Каждый инструмент принимает опциональный параметр `verbosity` (по умолчанию: `min`):

| Уровень | Описание | Поля карточки |
|---------|----------|---------------|
| `min` | Компактный, экономит контекст | 9 полей: id, title, url, board, column, owner, updated, asap, blocked |
| `normal` | Основные поля | ~22 поля: + даты, состояние, теги, участники, дорожка, тип, размер, дедлайн |
| `max` | Полный анализ | ~30 полей: + описание, чеклисты, блокеры, внешние ссылки |
| `raw` | Полный ответ API | Все поля, как их возвращает Kaiten API |

## Надёжность

- **Таймаут запросов:** настраиваемый HTTP-таймаут через `AbortController` (по умолчанию 10 сек). Предотвращает зависание при сетевых проблемах.
- **Автоматические повторы:** неудачные запросы (429, 408, 5xx, сетевые ошибки, таймауты) повторяются до 3 раз с экспоненциальной задержкой и джиттером. Заголовок `Retry-After` учитывается.
- **TTL-кэш:** пространства, доски, колонки, дорожки, типы карточек и пользователи кэшируются в памяти (по умолчанию 5 мин). Исключает лишние API-вызовы для справочных данных.
- **Валидация конфигурации:** все переменные окружения проверяются при старте через Zod. Некорректные значения выдают понятные сообщения об ошибках.
- **Обрезка ответов:** ответы длиннее 100k символов автоматически обрезаются для предотвращения переполнения контекста.
- **Защита от падений:** необработанные исключения и rejected промисы логируются в stderr без остановки сервера.

## Известные ограничения

- **Rate limits:** API Kaiten может ограничивать частоту запросов. Повторы обрабатывают временные 429-ошибки, но при устойчивой перегрузке нужно снижать частоту запросов.

## Решение проблем

- **Сервер не запускается:** проверьте, что `KAITEN_API_TOKEN` и `KAITEN_URL` заданы в `env` блоке конфигурации MCP.
- **Ошибки 401:** токен мог истечь или быть невалидным. Создайте новый в профиле Kaiten.
- **Ошибки 502:** инстанс Kaiten может быть временно недоступен. Сервер повторит запрос автоматически.
- **Большие ответы:** используйте фильтры (`boardId`, `spaceId`) или уменьшите `limit` для сокращения объёма ответа.
